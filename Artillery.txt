UI:
- дезактивировать все объекты, которые не нужны
Оптимизация:
	- при любых изменениях перерисовывается весь канвас. Поэтому статичные и динамичные элементы нужно размещать на разных канвасах
	- при отрисовке элементы обрабатываются пакетами. Пакеты рабиваются элементами, котрые имеют другой материал, или их граница перекрывает два других элемента, расположенных над или под. Поэтому нужно:
		- переместить в иерархии объекты, которые имеют один и тот же материал на соседние в иерархии уровни
		- переместить объекты так, чтобы они не перекрывались
	- по возможности применять простой шейдер для UI
	- заменять множество перекрывающихся элементов меньшим количеством более сложных (работает быстрее, но тратит больше памяти)
	- если UI не перекрывает всю сцену - переводить сцену в текстуру, и заменять ею объекты на фоне. При этом отключать камеру сцены.

Игра:
- ограниченное количество слотов оружия
- каждый тип оружия имеет максимальное количество боеприпасов

Быстрая игра:
- можно выбирать уровень противника без ограничений
- можно выбрать карту из открытых
- можно выбрать оружие из открытых
- количество слотов соответствует званию

Кампания:
Вначале два слота оружия. Каждый уровень - одна карта.
После боя:
- начисляются очки
- за очки можно открыть новый слот и новое оружие
После победы:
- доступна следующая карта
- уровень противника повышается
- начисляются доп очки (много)

Каждая карта - определенный набор оружия противника
